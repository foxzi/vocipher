# Configuration / Конфигурация

## English

Vocipher is configured entirely through environment variables. No config files needed.

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `VOCIPHER_ADDR` | `:8090` | HTTP server listen address (e.g. `:8080`, `127.0.0.1:8090`) |
| `VOCIPHER_DB_PATH` | `vocipher.db` | Path to SQLite database file |
| `VOCIPHER_TURN_IP` | *(disabled)* | Public IP address for the embedded TURN server. If not set, TURN is disabled and only STUN is used |

### Examples

**Development (defaults):**

```bash
make run
# Listens on :8090, DB at ./vocipher.db, no TURN
```

**Custom port:**

```bash
VOCIPHER_ADDR=:3000 ./vocipher
```

**With TURN enabled:**

```bash
VOCIPHER_TURN_IP=203.0.113.1 ./vocipher
# HTTP on :8090, TURN on UDP :3478
```

**Docker:**

```bash
docker compose up -d
# Configure via docker-compose.yaml environment section
```

### TURN Server

The embedded TURN server starts automatically when `VOCIPHER_TURN_IP` is set. It listens on UDP port 3478.

**When to enable TURN:**
- Users are behind corporate firewalls or symmetric NATs
- WebRTC connections fail with "ICE failed" errors
- The server is deployed on the public internet

**When TURN is NOT needed:**
- All users are on the same LAN
- Local development/testing

**TURN credentials** are generated automatically at each startup. A random shared secret is created, and the username `vocipher` is used. Credentials are passed to clients through the HTML template -- no manual configuration needed.

### ICE Configuration

The server provides ICE servers to clients via a JavaScript variable injected into the page:

```javascript
// Without TURN:
window.VOCIPHER_ICE_SERVERS = [
    {"urls": "stun:stun.l.google.com:19302"},
    {"urls": "stun:stun1.l.google.com:19302"}
];

// With TURN:
window.VOCIPHER_ICE_SERVERS = [
    {"urls": "stun:stun.l.google.com:19302"},
    {"urls": "stun:stun1.l.google.com:19302"},
    {"urls": "turn:203.0.113.1:3478", "username": "vocipher", "credential": "<auto-generated>"}
];
```

### Database

SQLite database is created automatically on first run. WAL mode is enabled for concurrent read/write performance. The database file location is controlled by `VOCIPHER_DB_PATH`.

**Docker:** The database is stored on a named volume (`vocipher-data`) at `/app/data/vocipher.db`, so data survives container restarts.

### Sessions

- Sessions expire after 30 days
- Expired sessions are cleaned up automatically every hour
- Session tokens are 64-character hex strings (32 bytes of entropy)

## Русский

Vocipher настраивается полностью через переменные окружения. Конфигурационные файлы не нужны.

### Переменные окружения

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `VOCIPHER_ADDR` | `:8090` | Адрес HTTP-сервера (например `:8080`, `127.0.0.1:8090`) |
| `VOCIPHER_DB_PATH` | `vocipher.db` | Путь к файлу SQLite базы данных |
| `VOCIPHER_TURN_IP` | *(выключен)* | Публичный IP для встроенного TURN-сервера. Если не задан, используется только STUN |

### Примеры

**Разработка (значения по умолчанию):**

```bash
make run
# Слушает :8090, БД в ./vocipher.db, без TURN
```

**Другой порт:**

```bash
VOCIPHER_ADDR=:3000 ./vocipher
```

**С включённым TURN:**

```bash
VOCIPHER_TURN_IP=203.0.113.1 ./vocipher
# HTTP на :8090, TURN на UDP :3478
```

### TURN-сервер

Встроенный TURN-сервер запускается автоматически при установке `VOCIPHER_TURN_IP`. Слушает на UDP порту 3478.

**Когда включать TURN:**
- Пользователи за корпоративными файрволами или symmetric NAT
- WebRTC-соединения падают с ошибкой "ICE failed"
- Сервер развёрнут в публичном интернете

**Когда TURN не нужен:**
- Все пользователи в одной локальной сети
- Локальная разработка/тестирование

**Credentials** генерируются автоматически при каждом запуске и передаются клиентам через HTML-шаблон.

### База данных

SQLite создаётся автоматически при первом запуске. WAL-режим включён для конкурентного чтения/записи. В Docker БД хранится в named volume.

### Сессии

- Сессии истекают через 30 дней
- Протухшие сессии автоматически очищаются каждый час
- Токены -- 64-символьные hex-строки (32 байта энтропии)
